TITLE
Tic Tac Toe Memory Test
GOAL
Build a simple web or mobile app that measures visual spatial working memory and recognition for symbol location patterns under time pressure, with increasing load and interference.
CORE TASK SUMMARY
The app briefly shows one or more tic tac toe style grids containing X and O symbols placed in specific cells. The grid disappears. The user must reconstruct the exact layout by tapping cells and choosing X or O for each selected cell. Difficulty increases by increasing the number of placed symbols, adding a second grid to remember, decreasing display time, adding delay, and adding interference via distractor shapes and decoy response options.
IMPORTANT CONSTRAINTS
1. Randomize layouts every trial.

2. Some trials show partially filled grids, not always full.

3. Two trials per level.

4. Clearly instruct that only X and O count. Distractor shapes are noise.

5. Record accuracy and reaction time.

6. Final results page shows raw performance and a percentile comparison only if norms exist. If no norms exist, show “local sample comparison” or “not available yet.”

PAGE 4: TEST LOOP
Run levels. Each level has exactly 2 trials.
Define these constants for shapes and visuals.
Grid appearance:
Background white or light neutral
Grid lines dark gray
Grid cell count depends on level, start at 3 by 3
Cell size: responsive, but keep square cells
Show grid centered
Symbols:
X drawn as two diagonal strokes
O drawn as a circle outline
Both black, thick line weight for visibility
Distractor shapes:
Triangle: equilateral outline
Square: outline
Star: simple 5 point outline
Diamond: outline rhombus
These distractors should appear only during the display phase, not counted in scoring.
Make distractors a different color if you want, but they must still be clearly shapes.
If you want to keep it simpler, make them gray and keep X and O black.
Interference rule:
When distractors are present, they can appear in empty cells or outside the grid margin.
They must never replace X or O. X and O placements stay the targets.
In instructions and UI labels, explicitly say ignore distractor shapes.
Delay rule:
After display, show a blank screen with a fixation cross for a delay that increases with difficulty.
Two grid rule:
When two grids are used, show Grid A then brief blank then Grid B.
Then reconstruction phase requires rebuilding Grid A first, then Grid B second.

TRIAL GENERATION RULES
For each grid in a trial:
         1. Create list of all cells for GridSize by GridSize.

         2. Randomly select NumTargets distinct cells as target positions.

         3. For each target cell assign a symbol:
Option A: 50 percent X and 50 percent O, random
Option B: balanced count when possible, for example if NumTargets is even, half X half O
Use Option B for cleaner scoring.

         4. Ensure no two targets overlap obviously, overlap is already prevented by distinct cells.

         5. For distractors:
Select NumDistractors cells from remaining empty cells or positions outside grid.
Choose distractor shape types randomly from Triangle, Star, Diamond, Square.
Distractors must not occupy target cells.

Special case:
Level 1 Trial 1 must be X in center cell only.

SCORING PER TRIAL
For each grid:
Target set is the exact mapping of cell to symbol.
Compute:
            1. CorrectPlacements:
Count cells where the user placed the correct symbol in the correct cell.

            2. CommissionErrors:
Count cells where user placed any symbol in a cell that should be empty, including X, O, or decoys.

            3. OmissionErrors:
Count targets that were not placed correctly, meaning either left blank or wrong symbol.

            4. GridAccuracyPercent:
CorrectPlacements divided by NumTargets times 100

TrialCorrectBinary:
A trial is correct if GridAccuracyPercent is at least 80 percent and CommissionErrors is 0
If you want stricter scoring, require 100 percent exact match. Pick one and keep it consistent.
Two grid trials:
Compute scores per grid and combine:
CombinedCorrectPlacements equals sum across grids.
CombinedNumTargets equals NumTargets times NumGrids.
CombinedAccuracyPercent equals combined correct divided by combined targets.
TrialCorrectBinary uses combined values.
Store all trial details for later review.

OVERALL SCORE
Compute these summary metrics:
               1. TotalCorrectPlacements across all trials

               2. TotalTargets across all trials

               3. OverallAccuracyPercent

               4. MeanReactionTimeMs

               5. LevelPassedCount: number of levels where at least 1 of 2 trials is TrialCorrectBinary true

               6. HighestLevelPassed: maximum level with pass criteria met

Primary score to show:
HighestLevelPassed and OverallAccuracyPercent
